# Contributing to AgentHub Issues

Thank you for helping improve AgentHub! This guide explains how to file effective issues.

## Before You Open an Issue

1. **Search existing issues** — your problem or idea may already be tracked. Check both [open](../../issues) and [closed](../../issues?q=is%3Aissue+is%3Aclosed) issues.
2. **Check the documentation** — the answer might already be in the product's guides or README.
3. **Identify the product** — know whether your issue is about **AgentHub Pro** or **AgentHub Analyst**.

## Writing a Good Bug Report

- **Use a clear title** — `[Bug]: Agent router skips registry when prompt contains special characters` is better than `[Bug]: routing broken`
- **Include steps to reproduce** — list the exact sequence of actions. Assume the reader has never seen the bug.
- **Include your environment** — OS, IDE/AI tool, AgentHub version (check the `VERSION` file)
- **Attach logs** — copy relevant terminal output or error messages. Use code blocks for formatting.
- **One bug per issue** — don't bundle multiple problems together.

## Writing a Good Feature Request

- **Describe the problem first** — explain the pain point before jumping to the solution.
- **Propose a solution** — describe what you'd like to happen. Be specific.
- **Mention alternatives** — if you've considered workarounds, list them.
- **Provide context** — include which agents or MCP tools are relevant.

## Asking a Question

- **Be specific** — "How do I configure the Jira MCP tool?" is better than "How do I set it up?"
- **Share what you've tried** — this helps us give a targeted answer instead of guessing.

## Labels

After you submit an issue, maintainers will apply labels:

| Label | Meaning |
|-------|---------|
| `product: pro` | Relates to AgentHub Pro |
| `product: analyst` | Relates to AgentHub Analyst |
| `bug` | Something is broken |
| `feature` | Feature request |
| `question` | Question or support |
| `triage` | Awaiting initial review |
| `confirmed` | Reproduced or accepted |
| `priority: critical/high/medium/low` | Severity level |

## Code of Conduct

All participants are expected to follow our [Code of Conduct](CODE_OF_CONDUCT.md). Be respectful, constructive, and patient.

## Security Issues

**Do not file security vulnerabilities as public issues.** Email us at **agenthub-ai-team@protonmail.com** instead.
