# AgentHub — Issue Tracker

This repository is the **public issue tracker** for all AgentHub products. Use it to report bugs, request features, or ask questions.

## Products

| Product | Description | Type |
|---------|-------------|------|
| **AgentHub Pro** | Deterministic agent orchestration system for AI coding assistants (Claude Code, GitHub Copilot, Cursor). Provides structured agents for bug investigation, requirements engineering, and documentation generation with MCP tool integrations. | Paid |
| **AgentHub Analyst** | Lightweight agent for bug investigation and feature exploration — the free edition of AgentHub. | Free |

## How to File an Issue

1. Go to [**New Issue**](../../issues/new/choose)
2. Choose a template:
   - **Bug Report** — something is broken or not working as expected
   - **Feature Request** — suggest a new capability or improvement
   - **Question / Support** — ask a question about usage, setup, or behavior
3. Fill in the template fields — the more detail you provide, the faster we can help
4. Select the correct **product** (AgentHub Pro or AgentHub Analyst) in the dropdown

## Labels

### Product
| Label | Description |
|-------|-------------|
| `product: pro` | Related to AgentHub Pro |
| `product: analyst` | Related to AgentHub Analyst |

### Type
| Label | Description |
|-------|-------------|
| `bug` | Something is broken |
| `feature` | New feature request |
| `question` | Usage question or support |
| `documentation` | Documentation improvement |

### Priority
| Label | Description |
|-------|-------------|
| `priority: critical` | Blocking issue, needs immediate attention |
| `priority: high` | Important, should be addressed soon |
| `priority: medium` | Normal priority |
| `priority: low` | Nice to have, non-urgent |

### Status
| Label | Description |
|-------|-------------|
| `triage` | Needs initial review |
| `confirmed` | Reproduced / accepted |
| `wontfix` | Will not be addressed |
| `duplicate` | Duplicate of another issue |

## Guidelines

- **Search first** — check [existing issues](../../issues) before creating a new one
- **One issue per report** — don't combine multiple bugs or requests in a single issue
- **Be specific** — include versions, steps to reproduce, and relevant logs
- See [CONTRIBUTING.md](CONTRIBUTING.md) for full guidelines
